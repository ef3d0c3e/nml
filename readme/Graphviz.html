1. Graphs from graphviz ðŸ”—

The Graphviz functionnality requires the dot executable. More information on Graphviz's website.

2. Synopsis ðŸ”—

Graphs blocks are delimited by [graph]...[/graph]

3. Properties ðŸ”—

  • layout The layout engine, defaults to dot see Graphviz's documentation. Allowed values:
    1. dot
    2. neato
    3. fdp
    4. sfdp
    5. circo
    6. twopi
    7. osage
    8. patchwork
  • width The resulting svg's width property, defaults to 100%

5. Graphiz cache ðŸ”—

Graphviz graphs that have been rendered to svg are stored in the cache database, under table cached_dot. Unless you modify the graph or it's properties, it won't be rendered again, instead it will be sourced from the database.

6. Bindigs ðŸ”—

  • nml.graphviz.push(layout, width, dot)
    • layout (string) the layout engine
    • width (string) the width property (empty string for default)
    • dot (string) the graphviz code
